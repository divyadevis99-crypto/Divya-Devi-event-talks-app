# BigQuery Release Notes Explorer

A beautiful, modern, single-page web application built using **Python Flask** and plain **vanilla HTML, JavaScript, and CSS** that aggregates, structures, and visualizes the latest Google BigQuery Release Notes, with a built-in Twitter (X) composer to share updates easily.

---

## 🌟 Key Features

* **Granular Release Cards**: Google Cloud release entries group all updates for a single day together. This app automatically parses the raw HTML and splits multi-update entries into distinct, individual cards (e.g. separate Feature, Issue, or Deprecation cards) for clear scanning.
* **Instant Search & Category Filtering**: 
  * **Text Search**: Real-time matching against titles, categories, and content strings.
  * **Category Badges**: Quick filter tabs to toggle between **All**, **Features**, **Issues**, **Deprecations**, and **Others**.
* **Integrated X (Twitter) Composer**: Click the Twitter icon on any update card to open a custom draft editor modal:
  * **Real-time Character Counter**: A live gauge (out of 280 characters) with a circular SVG progress ring that transitions from blue to amber to red as you approach the limit.
  * **Interactive Draft**: Pre-formats notes with dates, descriptions, hashtags (`#GoogleCloud #BigQuery`), and direct official links.
  * **Copy to Clipboard**: Quick copy actions with pop-up success toast notifications.
  * **Post Web Intent**: Seamlessly hands over the final text to X's composer interface.
* **Modern Interface**: Designed with a clean glassmorphism dark-theme layout using custom HSL colors, Google Fonts (Outfit & Plus Jakarta Sans), custom scrollbars, and micro-animations for hover states and loading transitions.

## 🚀 Quick Start Guide

### Prerequisites
Make sure you have **Python 3.12+** installed on your system.

### 1. Install Dependencies
Clone the repository, open a terminal in the project directory, and install the required Python packages:
```bash
pip install -r requirements.txt
```

### 2. Start the Server
Run the Flask server:
```bash
python app.py
```
By default, the server will start in debug mode on **`http://127.0.0.1:5000`**.

### 3. Open in Browser
Open your browser and navigate to:
👉 **[http://127.0.0.1:5000](http://127.0.0.1:5000)**

## 🔧 Behind the Scenes: How It Works

1. **Backend Fetching & XML Parsing**: In `app.py`, `urllib.request` requests the official Atom feed. Python's built-in `xml.etree.ElementTree` traverses the feed.
2. **Sub-Update Splitting**: We split multi-topic daily entries using regular expressions based on the standard `<h3>` heading selectors generated by Google's release parser.
3. **HTML to Markdown-style Text conversion**: Hyperlinks, bold words, and inline code are mapped to markdown styling (`[text](url)` $\rightarrow$ `text (url)` and `<code>` $\rightarrow$ `` ` ``) for Twitter compatibility.
4. **Client-side Filter Engine**: In `main.js`, all operations occur locally in the DOM for sub-millisecond sorting and searching.
